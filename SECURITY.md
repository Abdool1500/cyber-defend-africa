# Security Policy

## Reporting a vulnerability

If you believe you've found a security vulnerability in the Cyber Defend
Africa LTD platform, please report it privately rather than opening a
public GitHub issue. Email the details to the address listed on our
[Responsible Disclosure page](/responsible-disclosure/) once the site is
live, or contact the maintainers directly.

Please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce it
- Any relevant logs, screenshots, or proof-of-concept code

We'll acknowledge your report and work with you toward a timely resolution
before any public disclosure.

## Scope

This applies to the Cyber Defend Africa LTD web platform and its REST API.
It does not apply to third-party services we link to (e.g., the Google Form
used for waitlist signups).

## Supported versions

Only the `main` branch is actively maintained and receives security fixes.

## What we ask

- Give us a reasonable time to investigate and fix an issue before public
  disclosure.
- Don't access, modify, or delete data belonging to other users while
  testing.
- Don't run automated scanners or load tests against production without
  prior coordination.
